# Overview

In Module 01, we built an AI system using manually written business rules.

Example:

```text
IF Salary > ₹30,000

AND Credit Score > 700

↓

Approve Loan
```

This approach works well for small problems.

But imagine a bank processing **10 million loan applications**.

Writing and maintaining thousands of business rules becomes almost impossible.

This is where **Machine Learning (ML)** changes everything.

Instead of writing rules manually, we provide historical data, and the computer learns the patterns automatically.

---

# Why Learn Machine Learning?

Machine Learning is one of the most important skills for an AI Engineer.

It helps computers:

- Learn from historical data.
- Recognize patterns.
- Make predictions.
- Improve without manually changing business rules.

Machine Learning powers many applications that we use every day.

Examples:

- Loan Approval
- Spam Detection
- Face Recognition
- Fraud Detection
- Product Recommendations
- Medical Diagnosis
- Customer Churn Prediction

# Learning Philosophy

This module focuses on becoming an **AI Engineer**, not a Machine Learning researcher.

Our goal is to understand:

- What the algorithm does.
- When to use it.
- Why we choose it.
- How to train it.
- How to evaluate it.

We do **not** focus on complex mathematics.

---

# Key Takeaways

- Machine Learning is a subset of AI.
- Machine Learning learns from data.
- Different algorithms solve different problems.
- Model evaluation is as important as model training.
- Choosing the right algorithm is more important than memorizing formulas.
