# 🤔 Why Should You Learn This?

Imagine you work for a bank.

Every day, the bank receives:

```text
10,000 Loan Applications
```

Can a developer write thousands of IF-ELSE rules?

Maybe.

Can they maintain those rules forever?

No.

Business rules change every day.

Machine Learning solves this problem.

---

# 🧠 What is Machine Learning?

Machine Learning (ML) is a subset of Artificial Intelligence.

Instead of writing business rules,

we provide **historical data**.

The computer learns the patterns automatically.

Simply remember:

> **Rule-Based AI → Humans write rules.**

> **Machine Learning → Computer learns rules.**

# 💼 AI Engineer Note

A common misconception is:

> Machine Learning is just about choosing an algorithm.

Not true.

In real projects:

- Understanding the business problem
- Collecting good data
- Cleaning the data
- Evaluating the model

are often more important than the algorithm itself.

---

# ⚠ Common Mistakes

❌ Machine Learning is magic.

✔ Machine Learning learns from data.

---

❌ More data always means better results.

✔ Better quality data is more important than more data.

---

❌ Machine Learning replaces programmers.

✔ Machine Learning still requires software engineering.

# 🎤 Interview Questions

## Q1. What is Machine Learning?

Machine Learning is a subset of Artificial Intelligence that enables computers to learn patterns from historical data without being explicitly programmed.

---

## Q2. Why was Machine Learning introduced?

Because writing and maintaining thousands of business rules became difficult.

Machine Learning automatically learns patterns from historical data.

---

## Q3. Is Machine Learning a subset of AI?

Yes.

Artificial Intelligence is the broader field.

Machine Learning is one part of AI.
